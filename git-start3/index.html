注意事项

理论上，git日常用到的命令是 diff show fetch rebase pull push checkout commit status 等，这些命令都不会导致代码丢失，假如害怕代码丢失，可以预先commit一次，再进行修改，但切记

不可使用自己不熟悉的命令
任何命令，不要加上-f的强制参数，否则可能导致代码丢失
建议多使用命令行，不要使用图形界面操作
